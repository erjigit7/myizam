Распределения similarity (max по top-20):

Вывод: **жёсткого порога, отделяющего ловушки от честных, не существует**
(0.597 против 0.604 — бритвенный зазор, а ky-вопросы ниже ловушек). Рабочая
конфигурация: **MIN_SIMILARITY = 0.5** (отсекает явный мусор: 2 из 3 ловушек)
**+ LLM-отказ по правилу 3 промпта** (фраза «нет ответа» без маркеров → refused) —
вместе дают 3/3 ловушки при сохранении ответов на все честные вопросы.
Перекалибровать на живых логах после деплоя.

## Guard (shadow-режим)

Распределение по 23 неотказным ответам: p50 = 0.995, avg = 0.907, min = 0.003
(низкие значения — на кыргызских ответах qwen с ломаной грамматикой; это shadow, на ответ не влияет).

### Sanity: подмена фактов руками (`eval/guard_sanity.py`)

| Тип порчи | Пример | честный | испорченный | Δ |
|---|---|---|---|---|
| Цифра («28»→«45 дней») | ТК ст.68 | 0.996 | **0.008** | +0.99 |
| Цифра («70 и 56»→«30 и 14») | ТК ст.148 | 0.996 | **0.010** | +0.99 |
| Цифра («2 часа»→«6 часов») | ТК ст.108 | 0.996 | **0.006** | +0.99 |
| Число словом («трех»→«девяти месяцев») | ТК ст.24 | 0.996 | 0.996 | ~0 |
| Число словом («одной четверти»→«трех четвертей») | СК ст.86 | 0.996 | 0.996 | ~0 |
| Единица («в месяц»→«в квартал») | ТК ст.94 | 0.996 | 0.996 | ~0 |
| Цифра в ответе при числе-слове в контексте | ЖК ст.45 | **0.009** (ложная тревога) | 0.010 | ~0 |

**Главная находка:** ragguard-kg на юридическом домене работает как детектор
*цифровой консистентности* — цифра в ответе обязана буквально встречаться в контексте.
Подмену цифр ловит идеально (Δ+0.99), словесные числа и единицы измерения — не видит,
а честный ответ цифрой при числе-словом в законе даёт ложную тревогу. Прямое следствие
синтетических негативов датасета (программная подмена цифр) — ровно тот доменный сдвиг,
который заявлен в caveat ragguard. Это и есть обоснование shadow-режима на старте
и план дообучения на юр.домене (фаза 2): негативы со словесными числами + пары
«цифра↔словесное число».

## Caveats

1. Все LLM-зависимые метрики (citation, ky-качество, guard-скоры кыргызских ответов)
   получены на qwen2.5-7B — прод-модель gpt-4o-mini даст другой (лучший) профиль,
   особенно на кыргызском. Retrieval-метрики от LLM не зависят (кроме ky-моста).
2. Кыргызский текст qwen генерирует с зацикливаниями — ky-ответы в проде на qwen
   недопустимы, это dev-ограничение.
